################
Model workspace
################

After you **open a model** from the home page, Veda Online loads the **model workspace**. The URL looks like this (your site may use ``vedaonline.cloud`` or another host):

``https://vedaonline.cloud/ModelPage.aspx``

On that page you use the **Modules** row to switch between the main areas of the application: **Navigator**, **Browse**, **Items detail**, **Attribute Master**, **Run Manager**, **Results**, and **Reports**. **Main Menu** in the product is the strip that includes **Home**, **Modules**, search, and model information—see the first guide below for detail.

**Reading this manual**

The sections in this chapter follow that same structure, so you can jump from the app to the matching page:

* **Main Menu** — Home, Modules list, search, and model information.
* **Navigator** — Excel templates, sync, and file status.
* **Browse** — Tabular view of input data.
* **Items detail** — Topology and parameters for items.
* **Attribute Master** — Topology and input parameters for items (see also Items detail).
* **Run Manager** — Cases and runs.
* **Results** — Model output analysis.
* **Reports** — Reporting and views.

In the sidebar, these pages are **nested under Model workspace** so they stay grouped together and map clearly to the model page—not as unrelated top-level topics.
